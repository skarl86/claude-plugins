# CS 신고 처리 흐름

> 사용자 신고가 들어오면 어떻게 흘러가는지

---

## 들어가며

신고가 들어왔을 때 우리 팀이 어떻게 처리하는지 짧게 정리해봤어요. 신고 → 트리아지 → 수정 → 검증 → 머지의 5단계예요.

---

## 전체 흐름

```
[CS 신고] → [트리아지 봇] → [담당자 지정] → [수정 PR] → [검증] → [머지]
                ↓                                            ↓
           (자동 분류)                                    실패 시 PR로 회귀
```

여기서 검증이 실패하면 PR로 되돌아가서 수정을 다시 해요.

---

## 1단계 — 트리아지

자동 봇이 CS 신고 본문을 보고 카테고리를 분류해요.

- 결제 관련 → 결제팀
- 배송 관련 → 물류팀
- 기타 → 일반 큐

---

## 2단계 — 검증

수정이 끝나면 다음 4가지를 자동으로 검증합니다.

- format · lint · build
- 회귀 테스트 (같은 패턴의 과거 신고 50건)
- 보안 스캔
- 영향 범위 분석

---

## 마무리

자동화가 잘 돼 있어서 사람이 개입하는 시점은 트리아지 결과 확인 + 수정 PR 리뷰 두 번이에요.
