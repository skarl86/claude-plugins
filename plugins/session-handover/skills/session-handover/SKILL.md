# session-handover

`/compact` 후의 요약은 손실이 있고, 새 세션은 컨텍스트가 0이다. 다음 세션의 모델이
**탐색 없이 곧바로 작업을 재개**할 수 있도록, (1) 진실의 원천이 되는 핸드오버 문서와
(2) 그 문서를 가리키는 킥오프 프롬프트를 만든다.

## 산출물 1 — 핸드오버 문서

### 저장 위치 규칙

- 워크트리에서 작업 중이면: `<worktree>/docs/HANDOVER-<주제>.md` (워크트리와 함께
  이동/정리되도록 워크트리 안에 둔다)
- 본체 레포 작업이면: `<repo>/docs/<YYYY-MM-DD>-<주제>-handover.md`
- 레포 밖 작업이면: 작업 산출물 옆에 `HANDOVER.md`
- 항상 **절대경로**를 사용자에게 알려준다 (킥오프 프롬프트에 그대로 들어간다)

### 표준 포맷

아래 섹션 순서를 지킨다. 비는 섹션은 "해당 없음"으로 명시(생략하면 다음 세션이
누락인지 해당없음인지 구분 못 한다).

```markdown
# HANDOVER: <주제> (<YYYY-MM-DD>)

## 1. 작업 위치 (정확히)
- cwd / 워크트리 절대경로, 브랜치명
- 최종 커밋 해시 + **push 여부** (미push면 반드시 명시 — 가장 흔한 함정)
- dev 서버/프로세스: 기동 명령 + 포트 (예: `node_modules/.bin/next dev -p 3100`,
  포트 고정 이유가 있으면 같이 — "외부 API key의 referrer/origin 제한" 등)

## 2. 목표와 현황 (3줄 이내)
- 무엇을 위해, 어디까지 왔고, 지금 어느 단계인가

## 3. 완료된 것
- 검증까지 끝난 항목 (검증 방법 한 줄씩)

## 4. 남은 작업 (번호 단계, 실행 가능한 단위로)
- ① ② ③ … 각 단계에 파일 경로·명령 포함

## 5. 검증 방법
- 재개 후 "여전히 정상인지" 확인하는 명령 (테스트, E2E 하네스, curl 등)

## 6. 라이브 함정 / 건드리지 말 것
- prod 영향 요소, 절대 수정 금지 영역, 알려진 비결정성
- (예: "두 서비스가 서로 다른 외부 프로젝트를 쓴다 — A=…, B=…")

## 7. 자격증명·환경
- env 파일 위치와 받는 법 (시크릿 매니저/CLI의 env fetch 명령 등), 필요한 토큰명
  (값은 절대 기록 금지 — 위치/이름만)

## 8. 핵심 파일 경로
- 이번 작업의 중심 파일 5~10개

## 9. 다음 액션 옵션 (있으면)
- A/B/C 선택지와 추천
```

## 산출물 2 — 킥오프 프롬프트

사용자가 새 세션에 그대로 붙여넣을 한 덩어리. **코드블록**으로 제시한다.
구조 (사용자가 실제로 쓰는 패턴 그대로):

```
<프로젝트/작업명>을 이어서 진행한다. 먼저 다음 핸드오버 문서를 Read로 읽고 시작해:
<핸드오버 문서 절대경로>
(관련 메모리 파일이 있으면: 메모리 <파일명>.md 도 읽어라)

핵심 컨텍스트:
- 작업 위치: <워크트리/브랜치/커밋, push 여부>
- 현황: <2~3줄 상태 요약 — 문서 안 읽어도 방향이 잡히는 수준>
- 핵심 주의: <라이브 함정 중 최상위 1~2개>

다음 액션: <남은 작업 ①부터, 또는 "문서 §4를 따라 진행">
```

원칙: 프롬프트는 **요약**, 문서가 **진실의 원천**. 프롬프트에만 있는 정보가
생기지 않게 한다(문서에 다 넣고 프롬프트는 가리키기만).

## 메모리 연동 (선택)

작업이 여러 세션에 걸친 장기 프로젝트면 auto-memory 의 `project_<주제>.md` 를
갱신하고 `MEMORY.md` 인덱스에 등재한다. 핸드오버 문서는 "지금 상태", 메모리는
"프로젝트의 지속 사실"로 역할을 나눈다 — 둘을 중복 작성하지 말 것.

## 마무리

1. 두 산출물 경로를 보고하고, 킥오프 프롬프트를 코드블록으로 보여준다.
2. `/compact` (이어서 계속할 때) 또는 `/clear` + 새 세션(완전히 새로 시작할 때)을
   안내한다 — 실행은 사용자 몫.

## 함정

- **미push 커밋**: push 여부를 빠뜨리면 다음 세션이 원격만 보고 "작업이 사라졌다"고
  오판한다. 반드시 §1에 명시.
- **상대경로 금지**: 다음 세션의 cwd는 다를 수 있다. 모든 경로는 절대경로.
- **시크릿 값 기록 금지**: env 값·토큰을 문서에 복사하지 않는다. 위치와 이름만.
- 핸드오버 문서 작성 자체도 컨텍스트를 쓴다 — 컨텍스트가 거의 다 찼으면 §1·§4·§6
  핵심만 먼저 쓰고 나머지를 채운다.
